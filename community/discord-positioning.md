# Discord Positioning Plan

This is the next community step once a visible contribution trail exists.

The goal is not to walk into Discord and announce a brand.

The goal is to join the conversation as someone who is already contributing useful work.

## Timing

Good timing:

- once multiple PRs are open
- once you can point to specific contributions
- once you can speak from actual use rather than speculation

That threshold is now met.

## Positioning Rule

Lead with:

- what you contributed
- what you learned
- why you are using MemPalace seriously

Only then mention:

- CDD as the broader workflow or methodology context

## What To Say

Best shape:

1. say you have been actively contributing
2. link to one or two representative PRs
3. say you are exploring MemPalace for software-engineering memory
4. mention that this work also feeds into CDD
5. invite feedback, not applause

## What Not To Say

- do not lead with "I'm building a methodology called CDD"
- do not sound like you are trying to draft off MemPalace attention
- do not imply endorsement or partnership
- do not dump four links with no context

## Recommended First Discord Message

Use something close to this:

```text
Hey everyone - I've been using MemPalace pretty seriously as part of some local-first software engineering workflow experiments, and I've started contributing back upstream too.

So far I've opened a few PRs around:
- software engineering memory examples
- Windows cleanup / test reliability
- palace_graph test coverage
- general_extractor test coverage

Examples:
- https://github.com/milla-jovovich/mempalace/pull/285
- https://github.com/milla-jovovich/mempalace/pull/286
- https://github.com/milla-jovovich/mempalace/pull/289
- https://github.com/milla-jovovich/mempalace/pull/291

The broader thing I'm exploring is how structured local-first memory can support long-running software work, especially handoffs, decisions, and multi-LLM workflows over time.

That work also connects to a methodology I'm building called CDD, Context-Driven Development, but I'm here first and foremost as someone trying to make MemPalace more useful for real engineering memory.

Would love to compare notes with anyone else using it for project knowledge or coding workflows.
```

Opened a few PRs so far:

## If People Ask About CDD

Use a short answer:

```text
CDD is the broader workflow or methodology I'm building around structured context, validated execution, and feedback in software delivery. MemPalace fits into that naturally as a local-first memory layer.
```

## Best Outcome

Success is not hype.

Success is:

- maintainers and users recognize your name from useful PRs
- people ask follow-up questions
- you get into real conversation about engineering-memory use cases
- CDD comes up naturally as the larger frame behind the work
