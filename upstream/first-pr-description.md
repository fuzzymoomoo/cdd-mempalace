# Proposed PR Title

`docs: add software engineering memory example`

## Proposed PR Description

This PR adds a small example document showing how MemPalace can be used for software engineering memory.

The goal is to make one practical use case more concrete:

- preserving design and architecture context
- organizing decisions, incidents, and handoffs
- retrieving the right memory during planning, implementation, and validation

Why this shape:

- the repo already has an `examples/` directory for focused setup and workflow docs
- software engineering is a strong fit for MemPalace because so much important project knowledge now lives in conversations, notes, and scattered documents
- this example stays generic and local-first without introducing any external dependency or private workflow assumptions

What is included:

- one example project layout using wings, halls, and rooms
- examples of engineering artifacts that fit durable memory well
- a simple breakdown of useful retrieval by stage

No behavior changes. Docs only.
